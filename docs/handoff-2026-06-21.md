## 一、Gemma 3 / Gemma 4 開啟 Agentic Loop

### 現況

`Gemma3Client` 和 `Gemma4Client` 已存在於 `layer_e/llm_client.py`，但都只實作了 `generate()`，**未實作 `generate_with_tools()`**。`AgenticPipeline` 在 `agentic_pipeline.py:97` 呼叫 `self._llm.generate_with_tools()`，因此 Gemma 目前丟出 `NotImplementedError`，agentic loop 完全無法啟動。

唯一支援 tool calling 的是 `GPT41Client`（Azure OpenAI）。

### 已知的兩個 Bug（閱讀 Gateway 文件後發現）

**Bug 1：`Gemma3Client.generate()` 傳了 system role — Gateway 會拒絕**

Gateway 文件明確標注「Gemma 不支援 system role」，但現有程式碼：
```python
# layer_e/llm_client.py:76–82 — 這樣會被 gateway 拒絕
messages=[
    {"role": "system", "content": system},  # ← 錯誤
    {"role": "user", "content": user},
]
```
修法：將 system prompt 合併進 user message：
```python
messages=[{"role": "user", "content": f"{system}\n\n{user}"}]
```
`Gemma4Client`（`llm_client.py:146–156`）有相同問題。

**Bug 2：`Gemma4Client` 指向不存在的路由 `/gemma4/v1`**

Gateway 文件只列出 `/gemma2`（Gemma2-9B）和 `/gemma3`（Gemma3-27B），沒有 `/gemma4`。`Gemma4Client` 目前指向 `http://172.31.6.3:8080/gemma4/v1` — 這個端點不存在。需向維運確認是否有新部署，或移除 `Gemma4Client`。

### 需要做的事

**Step 1（先決）：修 Bug 1 — system role 問題**

修改 `Gemma3Client.generate()` 和 `Gemma4Client.generate()`，將 system/user 合併：
```python
def generate(self, system: str, user: str) -> dict:
    combined = f"{system}\n\n{user}" if system else user
    response = self._client.chat.completions.create(
        model="/model",
        messages=[{"role": "user", "content": combined}],
        temperature=0.0,
    )
    raw = response.choices[0].message.content
    return _parse_json_response(raw)
```

**Step 2：確認 Gateway server 是否啟用了 vLLM tool calling**

vLLM 的 function calling 需要在**啟動 server 時**加參數（不是 client 端設定）：
```bash
# server 啟動方式（需向維運確認是否有加這兩個 flag）
vllm serve /model \
  --enable-auto-tool-choice \
  --tool-call-parser gemma    # Gemma 專用 parser
```

測試 server 是否支援 tool calling：
```bash
curl http://172.31.6.3:8080/gemma3/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "/model",
    "messages": [{"role": "user", "content": "hello"}],
    "tools": [{"type": "function", "function": {"name": "test", "description": "test",
               "parameters": {"type": "object", "properties": {}}}}],
    "tool_choice": "auto"
  }'
# 若回應有 "tool_calls" 欄位 → server 支援
# 若只有純文字 content → server 未啟用，走 Step 3b
```

**Step 3a（server 支援 tool calling）：** 在 `Gemma3Client` 加上 `generate_with_tools()`，參考 `GPT41Client` 實作（`llm_client.py:114–135`），model 改為 `"/model"`。

**Step 3b（server 不支援）：** Prompt-based tool calling — 在 user message 中注入工具定義 JSON，讓模型輸出固定格式的 tool call 文字，再用 parser 解析。需修改 `AgenticPipeline` 支援此模式。

### Layer E Prompt 改善（本 session 末段）

**問題：** Retrieval 撈到 DCIS 特定指引段落，LLM 直接套用在非 DCIS 病人身上（事實性錯誤）。

**根本解法（進行中）：** 擴展 `enrich_units()` 支援段落 — 用 LLM 偵測「限定適用對象」，並在 `embedding_text` 加上 `[適用] DCIS（原位癌）患者` 前綴，使語意相似度自然降低，不依賴任何 rule-based 過濾。

**實作狀態：** `layer_b/enrichment.py` 已加入 `_PARAGRAPH_SYSTEM` prompt 和段落 enrichment 路徑；`scripts/test_agentic_query.py` 已串入 enrichment 步驟（有 cache：`retrieval_units_乳癌診療指引-2026年_enriched.json`）。enrichment 需對 1645 個 unit 各發一次 LLM 呼叫，**首次執行需 10–20 分鐘**。
